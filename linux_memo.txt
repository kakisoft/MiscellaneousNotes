インストール時

　主な違いは「推奨(recommends)パッケージ」の扱いです。apt-getは推奨パッケージを表示するだけに留めますが、aptitudeは推奨パッケージもインストールしようとします
　(たぶん提案(suggests)パッケージはインストールしません。推奨と提案の違いはこのエントリの質疑応答が参考になります)。

アンインストール時

　主な違いは必要のなくなったパッケージ(孤児(orphan)パッケージと呼ばれるらしい)の扱いです。apt-getもaptitudeもあるパッケージをインストールしたとき、必要なパッケージも自動的にインストールしてくれますが、インストールしたパッケージを削除したとき、apt-getはそのパッケージだけを、aptitudeは(/var/log/aptitudeのログを元に)不要になったパッケージも削除します。

CIFS
CIFS（英名 Common Internet File System 読み キフスまたはシーアイエフエス）とは、マイクロソフト社が開発した、WindowsOSのファイル共有で使用されるSMB（Server Message Block）プロトコルを拡張したものである。SMBは下位層のプロトコルとしてNetBIOSを利用していたが、CIFSはTCP/IPを利用してファイル共有を行う。
SMBでは、ネットワーク外のルーティングが出来なかったが、CIFSではWindows以外のOSやネットワーク外のアプリケーションを利用してファイル共有が可能なため、中、大規模のネットワークで活用されるようになった。Windows OSには標準搭載されている。
 またLinux、Solaris、Mac OS XなどのUNIX系OS向けにCIFSを使用するアプリケーションソフトの「Samba」が提供されており、フリーウェアとして無償で使用することができる。Sambaは1992年にオーストラリアのコンピュータープログラマであるアンドリュー・トリジェルによって開発されたソフトウェアである。


CIFS（英名 Common Internet File System 読み キフスまたはシーアイエフエス）とは、マイクロソフト社が開発した、
WindowsOSのファイル共有で使用されるSMB（Server Message Block）プロトコルを拡張したものである。


マウントとは、コンピュータに接続した周辺機器や外部記憶装置（あるいは、装置に挿入されたディスクなどの記録メディア）
をOSに認識させ、利用可能な状態にすること。 逆に、接続されている装置などをOSから切り離し、
取り外し可能な状態にすることを「アンマウント」（unmount）という。
